FlatRadar — 使用指南

跨平台监控 Holland2Stay、OurDomain、Xior 房源，实时通知，H2S 支持自动预订。

快速开始

1 Docker（推荐，适合 VPS）

cp .env.example .env && mkdir -p data logs logs/caddy
# 修改 Caddyfile，替换 your.domain.com 为你的域名
# 修改 .env，设置 WEB_PASSWORD
docker compose up -d

浏览器打开 https://your.domain.com，登录后在 Dashboard 点击启动监控。

2 macOS

从 Releases 下载最新 .dmg，拖入 Applications，双击启动。

3 Windows

从 Releases 下载最新 .zip，解压后双击 flatradar.exe。

4 从源码运行

pip install -r requirements.txt
cp .env.example .env
python web.py

浏览器打开 http://127.0.0.1:8088。

支持平台

FlatRadar 监控三个房源平台。每条房源上的平台标签（H2S / OD / XR）标识来源。

平台覆盖范围数据粒度自动预订
Holland2Stay (H2S)26 个荷兰城市单元级（具体房号、租金、面积）✅ 全 GraphQL 流程
OurDomain (OD)Amsterdam（Diemen + South-East）单元级（#6045、面积、楼层、朝向）❌ 仅通知
Xior (XR)荷兰 30 栋楼（15 城市）单元级（M1.30.53、面积、租金、押金）❌ 仅通知
提示：前往设置 → 平台启用或禁用平台，设置 → 城市选择要监控的楼盘。

Web 面板概览

Web 面板是控制中心，左侧栏可访问所有功能。

仪表盘

总览：数据库总房源、今日新增、今日变更、最近抓取时间。最新房源列表和近 48h 状态变更。每条房源带平台标签（H2S / OD / XR）。

点击启动监控开始抓取，运行中圆点变绿。

房源列表

浏览数据库全量房源。支持按状态、城市、平台、户型、最高租金、最小面积、合同、能耗等筛选。点击房源名跳转到源平台详情页。

地图

所有房源在交互地图上标点。绿=可预订，橙=抽签中，灰=已入住。标记显示平台来源。

日历

按月显示入住日期，可按城市和平台筛选。

统计

7/30/90 天可切换：新增趋势、变更趋势、城市/状态/平台分布、租金分布、上线时段。

通知推送

前往用户管理 → 点击用户 → 通知渠道。

Telegram 跨平台

  1. 在 @BotFather 创建 bot，复制 token
  2. 给你的 bot 发一条消息
  3. 访问 https://api.telegram.org/bot<TOKEN>/getUpdates 获取 Chat ID
  4. 填入 Bot Token 和 Chat ID

Email 跨平台

填入 SMTP 服务器信息。常见配置：

服务商SMTP 地址端口加密
Gmailsmtp.gmail.com587STARTTLS
Outlooksmtp.office365.com587STARTTLS

开启两步验证后请使用应用专用密码。

iMessage 仅 macOS

填入收件人手机号（如 +31612345678）或 Apple ID 邮箱。仅在 macOS 上且 Messages.app 已登录时可用。

WhatsApp Twilio 付费

需要 Twilio 账号，填入 Account SID、Auth Token 和 whatsapp:+... 号码。

iOS 推送通知

使用 FlatRadar iOS App 时，APNs 推送自动发送。支持中英双语——根据设备语言自动选择。

提示：点击发送测试通知可逐渠道验证。

自动预订（仅 Holland2Stay）

前往用户管理 → 点击用户 → 自动预订。

工作流程

当 H2S 房源变为可预订且匹配条件时：

  1. 登录 H2S 账号
  2. 创建新购物车
  3. 加入房源（addNewBooking）
  4. 下单（placeOrder）
  5. 生成直链付款 URL（idealCheckOut）
  6. 即时发送付款链接

不会自动付款。收到链接后需手动完成支付。

OurDomain 和 Xior 仅通知不预订——其预订流程受 reCAPTCHA 保护，暂未实现自动化。

配置项

  • 启用自动预订 — 开关
  • 试运行模式 — 建议先勾选，验证流程但不实际下单
  • H2S 账号邮箱与密码
  • 支付方式 — iDEAL（默认）/ Visa / Mastercard
  • 预订过滤条件 — 可设置比通知条件更严格的规则
注意：确认试运行模式验证通过后再关闭，避免误下单。

平台与城市

独立开关 H2S、OurDomain、Xior。每个平台可单独选择监控城市或楼盘。Xior 楼盘按城市分组。

修改后点击保存并生效热重载，无需重启。

常见问题

监控显示"已停止"

在仪表盘点启动监控。若无法启动，查看日志查看器或 docker compose logs h2s。

没有房源出现

  • 确认设置中已选择城市/楼盘
  • 确认平台已启用
  • 查看错误日志是否有 403 或网络错误

通知收不到

  • 确认用户已启用且通知已开启
  • 确认至少一个通知渠道配置了有效凭据
  • 点击发送测试通知逐个调试

403 被 Cloudflare 屏蔽

  1. 在 .env 中通过 HTTPS_PROXY 配置住宅代理
  2. 重启监控以更换 TLS 指纹
  3. 暂停几小时冷却
实用技巧：日志查看器（管理员 → 侧栏终端图标）支持 Monitor Log 和 Errors Log 切换、关键词搜索、行号和级别着色，排查问题第一站。
