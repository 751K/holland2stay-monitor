🏠 Holland2Stay 监控 — 使用指南

监控 Holland2Stay 新房源，第一时间推送通知，抢在别人之前自动预订。

快速开始

1 Docker（推荐，适合 VPS）

cp .env.example .env && mkdir -p data logs logs/caddy
# 编辑 Caddyfile → 将 your.domain.com 替换为你的域名
# 编辑 .env → 设置 WEB_PASSWORD
docker compose up -d

打开 https://你的域名，登录后在 Dashboard 点击 启动监控。

2 macOS

从 Releases 下载最新 .dmg，拖入 Applications，双击启动即可自动打开浏览器。

3 Windows

从 Releases 下载最新 .zip，解压后双击 h2s-monitor.exe。

4 从源码运行

pip install -r requirements.txt
cp .env.example .env
python web.py

浏览器打开 http://127.0.0.1:8088。

Web 面板概览

Web 面板是控制中心，所有功能从左侧边栏进入。

📊 仪表盘

一览视图：数据库房源总数、今日新增、今日状态变更、最近抓取时间。展示最新房源列表和近 48 小时状态变更记录。

点击 启动监控 开始抓取。监控运行时侧栏圆点变绿。

📋 房源列表

浏览数据库中所有房源。按 状态、城市（多选）、名称、最高租金、最小面积、合同类型、租客要求（多选）筛选。点击房源名称跳转到 Holland2Stay 详情页。

🗺️ 地图

所有房源在地图上展示。绿色 = 可直接预订，橙色 = 摇号中，灰色 = 不可预订。

如果有地址未解析，admin 可点击 解析地址 按钮触发地理编码。

📅 日历

按月历展示入住日期，按城市筛选。点击日期查看当天可入住的房源。

📈 统计

支持 7/30/90 天范围切换的图表：

  • 新增房源趋势 — 每日新增数量
  • 状态变更趋势 — 每日状态变化次数
  • 城市 & 状态分布 — 环形图
  • 租金分布 — 9 个价格区间柱状图
  • 房源上线时间分布 — 按小时统计，看出 H2S 哪个时间段集中放房

设置通知推送

进入 用户管理 → 点击用户 → 第 2 步：通知渠道。

Telegram 跨平台

  1. 向 @BotFather 创建 Bot，获取 Token
  2. 给你的 Bot 发任意一条消息
  3. 访问 https://api.telegram.org/bot<TOKEN>/getUpdates 获取 Chat ID
  4. 将 Bot Token 和 Chat ID 填入用户表单

Email 跨平台

填写 SMTP 服务器信息。常见配置：

邮箱SMTP 地址端口安全模式
Gmailsmtp.gmail.com587STARTTLS
Outlooksmtp.office365.com587STARTTLS
QQ 邮箱smtp.qq.com587STARTTLS

如果开启了二次验证，请使用 App Password（应用专用密码）而非登录密码。

iMessage 仅 macOS

填写收件人手机号（如 +31612345678）或 Apple ID 邮箱。仅当监控程序运行在 macOS 且 Messages.app 已登录时可用。

WhatsApp Twilio 付费

需要 Twilio 账号。填写 Account SID、Auth Token 以及 whatsapp:+... 格式的号码。

提示：表单底部点击 发送测试通知，逐渠道验证配置是否正确。

自动预订

进入 用户管理 → 点击用户 → 第 4 步：自动预订。

工作流程

当有房源变为 可直接预订 且符合你的自动预订过滤条件时，监控程序会：

  1. 登录你的 Holland2Stay 账号
  2. 创建全新购物车
  3. 将房源加入购物车（addNewBooking）
  4. 提交订单（placeOrder）
  5. 生成直链付款链接（idealCheckOut）
  6. 立即推送付款链接给你

不会自动付款。你收到付款链接后手动完成支付。

配置项

  • 开启自动预订 — 总开关
  • 试运行模式 — 推荐先开启：验证登录和购物车流程，不实际提交订单
  • H2S 账号邮箱 & 密码 — 你的 Holland2Stay 登录凭据
  • 支付方式 — iDEAL（默认，推荐）、Visa、Mastercard
  • 自动预订专用过滤 — 可选，比通知过滤更严格的条件

点击 从通知过滤复制 可一键同步通知过滤条件。

安全提醒：在确认登录和 SKU 解析正常之前，请保持试运行模式开启。确认无误后再关闭以提交真实订单。

通知过滤条件

限制每个用户接收通知的房源范围。进入 用户管理 → 点击用户 → 第 3 步。

过滤项示例说明
最高月租€1200只推送 ≤ €1200/月 的房源
最小面积25 m²只推送 ≥ 25 m² 的房源
最低楼层1排除地面层（0）
入住类型Single多选 — 任一匹配
户型Studio、1多选 — 任一匹配
城市Eindhoven、Amsterdam多选 — 任一匹配
片区Centrum依赖所选城市动态加载
合同类型Indefinite只推长租，排除短租
租客要求student only多选 — 任一匹配
标签/促销Short-stay多选 — 任一匹配

所有条件为 AND 关系（同时满足）。留空 = 不限制。

全局设置

进入 设置（仅 admin）。以下配置影响整个监控进程。

设置项默认值说明
常规轮询间隔300 秒（5 分钟）非高峰期的正常轮询间隔
高峰期最大间隔60 秒高峰期起始间隔（荷兰 08:30–10:00）
高峰期最小间隔15 秒自适应下限，轮询不会低于此值
高峰时间段08:30–10:00荷兰本地时间，仅工作日
抖动比例0.20±20% 随机波动，避免被识别为机器
监控城市—勾选要抓取的城市（多选）
日志级别INFO排查问题时改为 DEBUG

保存后点击 立即生效 热重载，无需重启。

常见问题

监控显示"未启动"

在 Dashboard 点击 启动监控。如果无法启动，查看 日志查看器（admin → 侧栏终端图标）或运行 docker compose logs h2s。

抓取不到房源

  • 确认 设置 中已勾选目标城市
  • 查看 日志查看器 → 错误日志，是否有 403 屏蔽或网络错误
  • 如果使用代理，验证代理是否生效：docker compose exec h2s python -c "import curl_cffi.requests as req; print(req.get('https://api.ipify.org').text)"

收不到通知

  • 确认用户已启用且通知开关已打开
  • 确认至少配置了一个有效渠道
  • 点击 发送测试通知 逐个渠道排查
  • 查看 日志查看器 → 错误日志 确认发送失败原因

自动预订失败

  • 验证 H2S 账号邮箱/密码是否正确（尝试手动登录）
  • 检查账号是否已有预留订单（登录 H2S 手动取消）
  • 先开启试运行模式验证流程，确认无误后再正式启用

403 被屏蔽（Cloudflare）

Holland2Stay 的 Cloudflare WAF 封了你的请求。解决方法：

  1. 在 .env 中配置 HTTPS_PROXY 使用住宅代理
  2. 重启监控获取新的 TLS 指纹
  3. 暂停监控几小时让屏蔽冷却
小技巧：日志查看器（admin → 侧栏终端图标）支持在运行日志和错误日志之间切换，有关键词搜索、行号和级别着色。排查问题时从这里开始最方便。
