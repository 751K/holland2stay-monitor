🏠 Holland2Stay Monitor — User Guide

Monitor new listings on Holland2Stay, get instant notifications, and auto-book before anyone else.

Quick Start

1 Docker (recommended for VPS)

cp .env.example .env && mkdir -p data logs logs/caddy
# Edit Caddyfile → replace your.domain.com with your domain
# Edit .env → set WEB_PASSWORD
docker compose up -d

Open https://your.domain.com, log in, then click Start monitor on the Dashboard.

2 macOS

Download the latest .dmg from Releases, drag to Applications, double-click. The browser opens automatically.

3 Windows

Download the latest .zip from Releases, extract and run h2s-monitor.exe.

4 Run from source

pip install -r requirements.txt
cp .env.example .env
python web.py

Open http://127.0.0.1:8088.

Web Panel Overview

The web panel is your control center. Everything is accessible from the left sidebar.

📊 Dashboard

At-a-glance view: total listings in database, new listings today, status changes today, last scrape time. Shows the newest listings and recent status changes (48h).

Click Start monitor to begin scraping. The dot turns green when the monitor is running.

📋 Listings

Browse all listings in the database. Filter by status, city (multi-select), name, max rent, min area, contract type, or tenant requirement (multi-select). Click any listing name to open its Holland2Stay page.

🗺️ Map

All listings plotted on an interactive map. Green = available to book, orange = lottery, grey = not available.

Click Geocode (admin only) to resolve addresses to coordinates if some markers are missing.

📅 Calendar

Month-by-month calendar showing move-in dates. Filter by city. Click a date to see all listings available that day.

📈 Statistics

Charts for 7/30/90-day ranges:

  • New listings trend — daily count over time
  • Status change trend — daily transitions
  • City & status distribution — doughnut charts
  • Rent distribution — 9 price buckets
  • Listing drop time — which hours H2S releases new listings (NL time)

Setting Up Notifications

Go to Users → click a user → Step 2: Notification Channels.

Telegram Cross-platform

  1. Create a bot with @BotFather and copy the token
  2. Send any message to your new bot
  3. Visit https://api.telegram.org/bot<TOKEN>/getUpdates to get your chat ID
  4. Paste the Bot Token and Chat ID into the user form

Email Cross-platform

Fill in SMTP server details. Common configurations:

ProviderSMTP HostPortSecurity
Gmailsmtp.gmail.com587STARTTLS
Outlooksmtp.office365.com587STARTTLS

Use an App Password (not your regular password) if 2FA is enabled.

iMessage macOS only

Enter the recipient's phone number (e.g. +31612345678) or Apple ID email. Only works when the monitor runs on macOS with Messages.app signed in.

WhatsApp Twilio paid

Requires a Twilio account. Fill in Account SID, Auth Token, and the whatsapp:+... numbers.

Tip: Click Send Test Notification at the bottom of the user form to verify each channel works before going live.

Auto-Booking

Go to Users → click a user → Step 4: Auto Booking.

How it works

When a listing becomes Available to book and matches your auto-booking filters, the monitor will:

  1. Log into your Holland2Stay account
  2. Create a fresh shopping cart
  3. Add the listing to cart (addNewBooking)
  4. Place the order (placeOrder)
  5. Generate a direct payment link (idealCheckOut)
  6. Send you the payment link immediately

It does NOT auto-pay. You receive the payment URL and complete payment manually.

Configuration

  • Enable auto-booking — toggle on
  • Dry run — recommended at first: validates login & cart without actually placing orders
  • H2S account email & password — your Holland2Stay login
  • Payment method — iDEAL (default), Visa, or Mastercard
  • Auto-booking filters — optional stricter rules (independent from notification filters)

Click Copy from notification filters to quickly duplicate your notification filter settings.

Safety: Keep Dry run on until you've verified that login and SKU resolution work correctly. Turn it off only when you're ready to place real orders.

Tips & Troubleshooting

Monitor shows "stopped"

Click Start monitor on the Dashboard. If it doesn't start, check the logs at Log Viewer (admin only) or docker compose logs h2s.

No listings appearing

  • Make sure cities are selected in Settings
  • Check Log Viewer → Errors Log for 403 blocks or network errors
  • If using a proxy, verify it's working: docker compose exec h2s python -c "import curl_cffi.requests as req; print(req.get('https://api.ipify.org').text)"

Notifications not arriving

  • Check the user is enabled and notifications are on
  • Verify at least one channel is configured with valid credentials
  • Click Send Test Notification to debug each channel
  • Check Log Viewer → Errors Log for send failures

Auto-booking failed

  • Verify your H2S email/password are correct (try logging in manually)
  • Check if your account already has a pending reservation (log into H2S and cancel it)
  • Enable Dry run first to validate the flow without placing real orders

403 Blocked (Cloudflare)

Holland2Stay's Cloudflare WAF is blocking requests. Solutions:

  1. Set up a residential proxy via HTTPS_PROXY in .env
  2. Restart the monitor to get a new TLS fingerprint
  3. Pause monitoring for a few hours to let the block cool off
Pro tip: The Log Viewer (admin → terminal icon in sidebar) lets you switch between Monitor Log (INFO+) and Errors Log (WARNING+) with keyword search, line numbers, and level color highlighting. Use it as your first stop for debugging.
